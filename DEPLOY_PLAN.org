* What Was Learned: Cloudflare Python Workers Reality (March 2026)

** Handler API: two incompatible modes

There are two distinct Python Worker APIs. They are incompatible and have
different size limits:

| Mode         | Handler              | Size limit (gzip) | Packages            |
|--------------+----------------------+-------------------+---------------------|
| Old          | =on_fetch(req, env)= | 64 MiB upload     | none (broken)       |
| New (active) | =WorkerEntrypoint=   | 3 MiB free        | vendored via pywrangler |
|              | class + =fetch=      | 10 MiB paid       |                     |

The old =on_fetch= handler is fully deprecated as of wrangler 4.71.0.
Only =WorkerEntrypoint= works.

** Package management: pywrangler, not wrangler

The new Python Workers use =pywrangler= for vendored packages.
With pure Python (no C extensions), no vendored packages are needed at all.
Worker bundle is ~80KB gzip.

** Size limits

| Resource             | Free plan | Paid ($5/mo) |
|----------------------+-----------+--------------|
| Worker gzip size     | 3 MiB     | 10 MiB       |
| Worker uncompressed  | 64 MiB    | 64 MiB       |
| CPU per request      | 10 ms     | 30 s         |
| Memory               | 128 MB    | 128 MB       |
| Requests             | 100K/day  | 10M/month    |
| KV reads             | 100K/day  | 10M/month    |
| KV writes            | 1K/day    | 1M/month     |

** CPU time: resolved

matplotlib in Pyodide WASM exceeded the 30-second CPU limit.
Pure Python SVG generation takes ~700ms total (including CSV parse + KV fetch).
Works comfortably on the paid plan.

** Memory: the real bottleneck with pure Python

Even without matplotlib, the 128MB limit is tight:
- Pyodide runtime baseline: ~70–80MB
- Full alarms CSV text (122K rows): ~12MB
- Python objects from CSV parsing: ~10MB
- Total without filtering: ~92–102MB → exceeds 128MB in practice

Fix: filter CSV to rows ≥ 2026-02-27 before storing in KV.
Filtered CSV (~9 days) is ~100KB. Total memory comfortable within 128MB.

KV cache key versioned as =csv:alarms:v3= to bust old over-sized entries.

** Response body: must be str, not bytes

Passing Python =bytes= to Cloudflare's =Response()= causes garbled output
(compressed/mangled first bytes in the HTTP response body).
Fix: =render_chart(...).decode("utf-8")= before passing to =Response()=.

** SVG in =<img>= vs =<object>=

SVG loaded via =<img src=...>= sandboxes the SVG — Google Fonts =@import=
does not load. Use =<object type="image/svg+xml" data=...>= instead.

** ZoneInfo / tzdata

=zoneinfo= requires the =tzdata= package in Pyodide.
Fix: moved =ISRAEL_TZ= out of =alarms_core.py= into =alarms_graph.py= (CLI only).

** seaborn / PIL

Not available in Pyodide. Both removed; SVG renderer needs neither.
