* Approach: Cloudflare Python Workers + Dual Runner

Cloudflare Python Workers run on Pyodide (CPython compiled to WASM).
Pyodide ships with *matplotlib* and *numpy* pre-built, so the
existing chart code can run with minimal changes.

The key insight: extract shared logic into =alarms_core.py= so both
the local CLI and the Cloudflare Worker can use it. The local script
stays unchanged in behavior.

* Architecture

#+begin_src
alarms_core.py       ← pure logic: CSV parsing, filtering, chart rendering
                        no I/O, no CLI, no caching — just functions
                        render_chart() returns bytes (PNG/SVG via BytesIO)

alarms_graph.py      ← local CLI runner (unchanged behavior)
                        argparse, requests, file cache, plt.show()
                        imports alarms_core for parsing + rendering

worker/
├── entry.py          ← Cloudflare Worker entry point
│                       query params, js.fetch, KV cache
│                       imports alarms_core for parsing + rendering
├── wrangler.toml
└── requirements.txt
#+end_src

* What Changes vs What Stays

** Stays the same
- =alarms_graph.py= CLI behavior: same args, same output, same caching
- All chart rendering logic (matplotlib, seaborn, colors, layout)
- CSV parsing, filtering, deduplication

** Changes
| Current                  | Cloudflare equivalent              |
|--------------------------+------------------------------------|
| =requests.get()=         | =fetch()= via Pyodide's =js= module |
| File-based cache         | Cloudflare KV with TTL             |
| =argparse= CLI args      | URL query parameters               |
| =plt.savefig(file)=      | =plt.savefig(BytesIO)= → Response  |
| =plt.show()=             | removed (worker only)              |
| =cities.csv= local file  | Bundled as Python dict             |
| Local font (ET-Book)     | Fall back to web-safe serif        |

** Step 3: Worker entry point

#+begin_src python
# worker/entry.py
from js import Response, Headers, URL, fetch as js_fetch
from pyodide.ffi import to_js
import json

from alarms_core import (
    load_alerts, load_api_alerts, render_chart,
    CITY_TRANSLATIONS, DEFAULT_AREA_FILTER, DEFAULT_BIN_HOURS, DEFAULT_START,
    ALARMS_CSV_URL, TZEVAADOM_API_URL,
)

async def fetch_csv(env):
    cached = await env.CACHE.get("csv:alarms")
    if cached:
        meta = await env.CACHE.get("csv:meta")
        return cached, meta or ""

    resp = await js_fetch(ALARMS_CSV_URL)
    text = await resp.text()
    last_mod = resp.headers.get("Last-Modified") or ""

    await env.CACHE.put("csv:alarms", text,
                        to_js({"expirationTtl": 30 * 60}))
    await env.CACHE.put("csv:meta", last_mod,
                        to_js({"expirationTtl": 30 * 60}))
    return text, last_mod

async def fetch_api_data(env):
    cached = await env.CACHE.get("api:alerts")
    if cached:
        return json.loads(cached)

    resp = await js_fetch(TZEVAADOM_API_URL)
    text = await resp.text()
    await env.CACHE.put("api:alerts", text,
                        to_js({"expirationTtl": 2 * 60}))
    return json.loads(text)

async def on_fetch(request, env):
    url = URL.new(request.url)
    path = url.pathname

    if path == "/" or path == "":
        return serve_landing_page()

    params = url.searchParams
    area = params.get("area") or DEFAULT_AREA_FILTER
    label = params.get("label") or CITY_TRANSLATIONS.get(area, area or "All Areas")
    start = params.get("start") or DEFAULT_START
    style = params.get("style") or "lines"
    threat = int(params.get("threat") or "0")
    bin_hours = int(params.get("bin_hours") or "1")
    fmt = "svg" if path.endswith(".svg") else "png"

    csv_text, data_cutoff = await fetch_csv(env)
    times, seen_ids = load_alerts(csv_text, area, threat, start)

    api_data = await fetch_api_data(env)
    api_times = load_api_alerts(api_data, area, threat, start, seen_ids)
    times = sorted(times + api_times)

    img_bytes = render_chart(times, label, bin_hours, start,
                             data_cutoff, style, fmt)

    content_type = "image/svg+xml" if fmt == "svg" else "image/png"
    headers = Headers.new(to_js({
        "Content-Type": content_type,
        "Cache-Control": "public, max-age=120",
    }))
    return Response.new(to_js(img_bytes), headers=headers)

* Risks and Mitigations

** 1. Cold start latency (3–10s)
Loading matplotlib WASM on first request is slow.
- Acceptable for this use case (not latency-critical)
- Cloudflare keeps workers warm between requests
- Can add a cron trigger (=scheduled= handler) to keep warm

** 2. CPU time limits
Free tier: 10ms CPU — *not enough* for matplotlib.
Paid ($5/mo): 30s CPU — fine.
- Start with paid plan, or consider pure SVG string builder later
  to drop back to free tier

** 3. Memory limits (128 MB)
matplotlib + 4MB CSV could be tight.
- Parse CSV line-by-line, pre-filter by date string prefix
- Cache parsed results in KV keyed by =(area, threat, start)=

** 4. seaborn not in Pyodide
- Addressed in Step 6: graceful fallback to matplotlib equivalents
- Visual output is nearly identical

** 5. No ET-Book font
- Falls back to Georgia / Palatino / DejaVu Serif
- Acceptable visual difference

** 6. ZoneInfo in Pyodide
- =zoneinfo= should work since Pyodide includes tzdata
- If not: hardcode Israel DST rules (simple)
