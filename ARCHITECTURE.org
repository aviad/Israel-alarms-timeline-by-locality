* Architecture

** Pure-Python modules (CLI + executable spec)

These stay Python for =alarms_graph.py= CLI and serve as the reference
implementation against which the Rust port was validated.

*** =alarms_core.py= — shared pure-Python logic (~391 lines)
- =render_chart(...)= → SVG string — pure string building, no matplotlib
  - Tufte-style: EB Garamond font, night shading, lines/dots modes
  - Forecast annotation: economist-style curved line + label
  - Night mode (8pm–6am): switches label to "tonight (until 7am)"
- =compute_prediction(...)= — extracted for header/desc embedding
- No external dependencies — pure stdlib

*** =city_translations.py= — city name mapping
- =CITY_TRANSLATIONS= — 1627 Hebrew→English city names

*** =israel_time.py= — Israel time utilities
- =_israel_utc_offset()=, =_epoch_to_israel()= — Israel DST (last Friday of March rule)

*** =data_loading.py= — CSV and API loading
- =load_alerts(csv_text, area_filter, threat, start)= → =(times, seen_ids)=
- =load_api_alerts(api_data, area_filter, threat, start, seen_ids)= → =times=
- =load_alerts_rich= / =load_api_alerts_rich= — event-level records for Ridge model
- =ROCKET_DESC= constant

*** =forecast.py= — prediction models (pure Python)
- =predict_remaining(times, now, method)= — "simple" (3-feature linear) or "advanced" (rate-sub blend)
  - Blend: w×rate_sub + (1-w)×rate, w=hours_remaining/24; MAE 1.54
- =predict_remaining_ridge(all_records, city, now, alpha=10)= — 34-feature Ridge regression
- =predict_night_rolling(times, now)= — 7-night rolling average (8pm–midnight fallback)
- =predict_night_ridge(all_records, city, now)= — 7am–7am Ridge (midnight–6am)
- Night mode selection: Ridge beats naive by 35% after midnight; rolling avg before midnight

*** =alarms_graph.py= — CLI wrapper
- argparse, requests, file-based cache
- PNG export via cairosvg with embedded ETBembo fonts as base64
- =--forecast off/simple/advanced/ridge=

** Rust Worker (=worker-rs/=) — the live worker

*** =src/entry.rs= — Worker entrypoint
- =#[event(fetch)]= handler; manual routing (match on path string).
- Routes: =GET /= (HTML), =GET /chart.svg=, =GET /chart.png=, =GET /healthz=, 404 fallback.
- Landing page: =include_str!("../assets/landing.html")= + placeholder substitution
  (=__DEFAULT_AREA__=, =__DEFAULT_START__=, =__CITIES_JSON__=).
- Two-phase loading handled client-side (JS calls =/chart.svg= twice — forecast=off, then full).
- KV key prefix =rs:= (=rs:csv:alarms:v1= 30-min TTL, =rs:api:v1:{epoch_min}= 2-min TTL).
- CSV streaming: full body fetched, filtered to rows ≥2026-02-27 before KV store.
- Hebrew city names decoded via =url_decode= (collects bytes before UTF-8 conversion).

*** =src/kv_cache.rs= — KV wrapper
- =get_text/put_text=, =get_bytes/put_bytes=, =get_zlib/put_zlib=.
- Zlib+base64: compressed bytes stored as KV text — avoids JSON-array corruption in local dev.

*** =src/svg.rs= — SVG renderer (port of =alarms_core.py=)
- =render_chart(RenderParams)= → =Result<String, String>=.
- Numeric format strings match Python exactly (=:.1=, =:.2=, =:.0=).
- Night shading, lines/dots, Tufte ticks, forecast curve, =<desc id="pred-data">= block.

*** =src/forecast.rs= — prediction models (port of =forecast.py=)
- All four models: simple/advanced, ridge, night-rolling, night-ridge.
- =solve_normal_equation= (Gauss-Jordan): β = (XᵀX + αI)⁻¹ Xᵀy.
- Insertion-order-preserving =city_rank_pct= (Vec alongside HashMap) to match Python dict order.

*** =src/data_loading.rs= — CSV/API loading (port of =data_loading.py=)
- =IsraelTime= struct (Ord-derived, parseable, epoch-constructable).
- All four load functions with full filter parity.

*** =src/israel_time.rs= — Israel time (port of =israel_time.py=)
- =epoch_to_israel=, =ymd_to_epoch=, =days_to_ymd=; DST last Friday of March.

*** =src/city_translations.rs= — city map (generated by =tools/gen_city_translations.py=)
- Sorted =&[(&str,&str)]= static slice; binary search lookup; 1627 entries.

* Free Plan Consideration

The Rust worker's 2ms startup and ~261KB bundle puts it well within
free plan CPU limits (10ms/request). If daily requests stay under
100k, downgrading to the free plan is viable. Watch for:
- CPU spikes on Ridge forecast for large areas (many records to train on).
- The free plan has a 10ms CPU limit *per request* — not wall-clock time.
  Ridge on a large city (e.g. תל אביב) may exceed 10ms CPU.
- Test with =wrangler tail= to observe actual CPU usage before downgrading.

* Lessons Learned

** Python Worker (historical)
- =WorkerEntrypoint= only (=on_fetch= deprecated since wrangler 4.71.0)
- =Response(bytes)= → garbled; must decode to =str= first
- SVG in =<img>= blocks =@import=; use =<object>= instead
- =zoneinfo= needs =tzdata= in Pyodide; moved to CLI only
- =datetime.fromtimestamp()= uses system TZ (UTC on Cloudflare); use explicit conversion
- KV.put with Python string → JS UTF-16 string doubles memory; use binary (Uint8Array)
- JavaScript =null= from KV.get is a JsProxy in Pyodide, NOT Python =None=

** Rust Worker
- Israel DST: last *Friday* of March (not Sunday like EU)
- wrangler local KV stores =Vec<u8>= as JSON array of integers, not raw binary;
  fix: base64-encode zlib output and store as KV text
- =city_rank_pct= requires insertion-order-preserving key tracking (Vec + HashMap)
  to match Python dict tiebreaking; plain HashMap iteration is non-deterministic
- Canvas sandboxes =<object>= SVGs; fonts for PNG export must be base64-embedded client-side
