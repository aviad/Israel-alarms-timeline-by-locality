#+title: Worker Migration Plan: Python (Pyodide) → Rust (workers-rs)
#+date: <2026-04-14 Tue>
#+author: Aviad

* Goal & Motivation

Migrate the Cloudflare Worker at =worker/entry.py= from Python (Pyodide) to
Rust (=workers-rs=) for performance and resource headroom.

** Why
- Pyodide cold start is ~300–500ms; Rust/WASM cold start is typically <10ms.
- Current per-request CPU ~700ms; expect 5–20× speedup on SVG build,
  CSV parse, and Ridge regression.
- 128MB memory ceiling has *already* forced workarounds (binary KV
  storage, zlib compression, pre-filtering CSV during stream). Rust
  removes the JS UTF-16 doubling problem entirely and uses a fraction
  of the memory.
- Bundle size: ~80KB Pyodide bundle but pulls full Pyodide runtime;
  Rust WASM is self-contained, often <500KB total deployed.
- Eliminates Pyodide-specific footguns (=JsProxy= vs =None=,
  =Response(bytes)= encoding bug, =zoneinfo/tzdata= absence).

** Non-goals
- Do *not* migrate =alarms_graph.py= CLI or the pure-Python modules used
  by it. Those stay Python for local/notebook use.
- No new features. Behavior parity first; optimization second.

* Target Stack

- =workers-rs= (=worker= crate) for the Worker entrypoint, KV, fetch.
- =wrangler= with =build.command = "cargo install -q worker-build && worker-build --release"=.
- =compatibility_date= unchanged; remove =python_workers= flag.
- Rust 1.78+, =wasm32-unknown-unknown= target.

** Crate choices
| Concern         | Crate                                    |
|-----------------+------------------------------------------|
| Worker runtime  | =worker= (Cloudflare official)             |
| HTTP fetch      | =worker::Fetch= (no =reqwest=, no tokio runtime) |
| CSV parsing     | =csv= (streaming, zero-copy where possible) |
| Compression     | =flate2= (zlib for KV payloads, parity with current) |
| Dates/times     | =time= crate (smaller than =chrono=, no tz db needed) |
| Linear algebra  | hand-rolled — Ridge is 34 features, do not pull =ndarray=/=nalgebra= |
| HTML escaping   | tiny inline helper or =askama_escape= |
| Templating (HTML landing page) | =askama= OR raw =format!= / =include_str!= with placeholders |
| Serialization   | =serde= + =serde_json= (only where needed at JS boundary) |

Avoid: =tokio=, =reqwest=, =chrono-tz=, =polars=, =ndarray=. They bloat WASM.

* Migration Strategy

** Phased, behavior-preserving
1. *Scaffold* =worker-rs/= alongside =worker/=, deploy under a
   different worker name (=alarms-rs=) and route. Keep Python live.
2. *Port pure logic first* (Israel time, city map, SVG render),
   covered by Rust unit tests that compare byte-for-byte against
   golden SVGs generated by the current Python.
3. *Port data loading* (CSV streaming + filter ≥ 2026-02-27 + threat
   filter). Test against a captured CSV fixture; assert identical
   =(times, seen_ids)= output.
4. *Port forecast models*. Ridge is the risky one — compare predictions
   to Python within tolerance (1e-6 absolute on coefficients trained
   on identical fixtures).
5. *Port worker entrypoint*: routing, KV cache, two-phase loading,
   landing page HTML, deeplinks.
6. *Cutover*: rename, swap routes, retire Python worker. Keep Python
   sources for CLI and as the executable spec for forecasting.

** Parity harness (critical)
Before declaring any module "done", run a parity test:
- Generate N=200 random parameter sets (city, date, threat, style,
  forecast mode).
- Hit Python worker locally (=wrangler dev=) and Rust worker locally.
- Diff SVG output. SVG must be byte-identical *or* deviations must be
  whitelisted (e.g. floating-point formatting in path coordinates —
  decide a canonical format and apply to both sides).
- Forecast numbers: relative error ≤ 1e-4.

This harness lives in =tools/parity/= as a small Python script that
spawns both wranglers and diffs.

* Risk Map

| Risk                                                              | Severity | Mitigation |
|-------------------------------------------------------------------+----------+------------|
| SVG output drift (float formatting, path rounding)               | HIGH     | Pin formatter (=format!("{:.2}", x)=) on both sides; add parity tests early |
| Ridge regression numerical drift (matrix inversion vs scikit-style closed form) | HIGH     | Implement closed-form ridge: =β = (XᵀX + αI)⁻¹ Xᵀy=; test on fixed fixture; compare to Python =forecast.py= within 1e-4 |
| Israel DST rule (last Friday of March, not Sunday) re-implemented incorrectly | MED      | Port =israel_time.py= verbatim; unit test boundary days for 2024–2027 |
| =city_translations.py= 1627-entry dict — translate to Rust       | LOW      | Generate =phf_map!= via build.rs from the Python source, OR embed as static =&[(&str,&str)]= sorted slice with binary search |
| KV binary format compatibility (existing zlib CSV cache)         | MED      | Use same zlib level + framing; on first deploy invalidate cache key prefix to avoid mixed reads |
| =Fetch= API differences for streaming CSV (~13.5MB)              | MED      | Use =worker::Fetch::Request= + =Response::stream()=; chunk-decode and filter rows incrementally; never materialize full body |
| HTML landing page (640 lines, embedded JS, mobile CSS)           | LOW      | Move to =include_str!("landing.html")= unchanged; only template the dynamic bits |
| WASM bundle size growth past 10MB paid limit                     | LOW      | Build with =opt-level="z"=, =lto=true=, =codegen-units=1=, =strip=true=; run =wasm-opt -Oz=; budget <1MB |
| Embedded fonts (EB Garamond, ETBembo) for PNG export             | LOW      | These live client-side already; no Rust change |
| Two-phase loading (chart now, forecast async)                    | MED      | Preserve the existing JS contract: same endpoints, same JSON shapes for the forecast overlay request |

* Module-by-Module Port Notes

** israel_time.rs
- Pure function, no deps. Implement =israel_utc_offset(epoch: i64) -> i32=
  using the time crate to get month/day/weekday for the Israel-local
  guess, then apply the last-Friday-of-March / last-Sunday-of-October
  rule (verify October rule against current Python).
- Test: cross 2026-03-27 (Fri) and 2026-10-25 boundaries.

** city_translations.rs
- Generate at build time from =city_translations.py= via a small Rust
  =build.rs= that parses the dict literal (or, simpler, run a one-shot
  Python script to emit a =.rs= file with =static MAP: phf::Map<&str,&str> = phf_map! {...}=).
- 1627 entries × ~30 bytes ≈ 50KB compiled.

** data_loading.rs
- Streaming CSV reader. Filter rows by =date >= start= and threat
  match during parse. Return =(Vec<i64> times, HashSet<String> seen_ids)=.
- Match Python =load_alerts= semantics exactly (same date parsing,
  same threat normalization).
- Same for =load_api_alerts= (JSON input via =serde_json::from_slice=).
- =_rich= variants: return =Vec<EventRecord>= for Ridge.

** forecast.rs
- Port four functions: =predict_remaining=, =predict_remaining_ridge=,
  =predict_night_rolling=, =predict_night_ridge=.
- Ridge: implement Gauss-Jordan or Cholesky on a 34×34 matrix (small,
  no SIMD needed). Add =#[cfg(test)]= comparison against a fixture
  generated by the Python implementation.

** alarms_core.rs (SVG)
- Pure string building. Use =std::fmt::Write= into a =String=.
- Define a =FormatF= helper that formats coordinates with a fixed
  precision matching Python's =repr=/format choices used in the
  current code. *Audit every numeric format string in alarms_core.py*
  and replicate.
- Night shading, lines/dots modes, Tufte axis ticks, forecast curve,
  =<desc id="pred-data">= block.

** entry.rs (worker entrypoint)
- =#[event(fetch)]= handler with router (=worker::Router=).
- KV cache: =env.kv("CACHE")?=. Reuse existing key names *or* prefix
  with =rs:= for a clean cutover.
- CSV cache: store =flate2::write::ZlibEncoder= output as bytes.
  Decompress on read.
- Two-phase loading: =/chart= returns SVG quickly with no forecast;
  client JS calls =/forecast= which returns JSON.
- Landing page: =include_str!= the existing HTML; only inject CSRF/
  version stamp if any (probably none).

* Test Plan (granular, per module)

Principle: every Rust module ships with (a) Rust unit tests against
fixtures, and (b) a cross-language parity test that pins Rust output
to Python output on the *same* input. Build fixtures once by running
the Python code and committing its output as the golden file.

** Fixture generation (do this first, day 1)
Write =tools/gen_fixtures.py= that runs the current Python modules on
a curated set of inputs and emits deterministic JSON/CSV/SVG into
=worker-rs/tests/fixtures/=. Re-runnable, committed. Fixtures:

- =alerts_mini.csv= — ~200 rows spanning 3 days, 5 cities, both threats
- =alerts_full.csv= — real CSV snapshot from one production pull
- =api_sample.json= — recent API response
- =ridge_training_set.json= — =all_records= for one city, one =now=
- =expected_times_*.json= — output of =load_alerts= per filter combo
- =expected_forecast_*.json= — output of each forecast function
- =expected_svg_*.svg= — golden SVGs for ~20 parameter combos
- =israel_offset_boundaries.json= — offsets on DST transition dates

** israel_time.rs — unit only, no fixtures needed
- DST-on boundary: last Friday of March 2024, 2025, 2026, 2027.
- DST-off boundary: last Sunday of October (verify rule first).
- Midsummer (definite +3), midwinter (definite +2).
- Epoch round-trip: =epoch → israel local → epoch= preserves value.

** city_translations.rs
- All 1627 keys present (assert =len == 1627=).
- Spot-check 10 known translations.
- Unknown key returns =None=, not panic.
- Build-time regen test: diff generated =.rs= against committed copy
  in CI; fail if Python source changed without regen.

** data_loading.rs — fixture-driven parity
Per filter combination (threat × city × start-date), assert:
- =times= vector equals Python output exactly (same length, same order, same values).
- =seen_ids= set equality.
- Edge cases as separate tests:
  - Empty CSV → empty result, no panic.
  - CSV with only header → empty result.
  - Row with malformed date → skipped (match Python behavior; verify first).
  - Unicode city names (Hebrew) round-trip through filter.
  - =start= exactly matching a row timestamp (inclusive vs exclusive — pin it).
  - Large CSV streamed in 64KB chunks vs all-at-once → identical output.
- Same suite for =load_api_alerts= against =api_sample.json=.
- =_rich= variants: compare record-by-record with =serde_json= equality.

** forecast.rs — numerical parity with tolerance
- =predict_remaining= "simple" and "advanced":
  - 10 fixture cases; assert within 1e-9 (pure arithmetic, should match exactly modulo f64 ordering).
  - Degenerate inputs: empty =times=, =now= before first alert, =now= after midnight.
- =predict_remaining_ridge=:
  - Fixture with frozen =all_records= + =now=; assert within 1e-4 of Python.
  - *Also* assert intermediate quantities match: feature vector
    (34 floats), XᵀX matrix (34×34), coefficients (34). Pin these
    to =ridge_intermediates.json= so drift is localized.
  - Singular/near-singular matrix case → verify regularization path.
- =predict_night_rolling=:
  - 7-night window with exactly 7 nights of data, <7, >7.
  - Transition at midnight (fallback change).
- =predict_night_ridge=: fixture parity + insufficient-data fallback.
- Property tests (=proptest=): prediction is non-negative, monotonic
  in rate where expected.

** svg.rs (alarms_core)
Byte-equality is the goal but brittle. Layered tests:

1. *Component tests* — call each internal helper (axis tick builder,
   night shading rect, forecast curve path) and assert the exact
   string fragment. Small, fast, localizes drift.
2. *Numeric format audit test* — a single test that renders a chart
   with known inputs and greps the SVG for all numeric literals,
   asserting each matches Python's formatting exactly. Catches the
   "is it =12.3= or =12.30=?" class of bug in one place.
3. *Golden SVG parity* — ~20 committed =.svg= fixtures; diff textually.
4. *Structural parity* (fallback if #3 too brittle) — parse both SVGs,
   compare element tree: same elements in same order, attributes
   equal, numeric attributes within 1e-6.
5. *Visual smoke* — render golden SVGs to PNG via =resvg= in a dev-
   dependency test, hash the PNG, commit hashes. Catches rendering
   regressions that textual diff might miss.

** kv_cache.rs
- Round-trip: put bytes, get bytes, equal.
- Zlib wrapper: compress → decompress → equal; reject corrupt payload
  with =Err=, not panic.
- Compatibility test: feed a zlib blob produced by the Python worker
  (captured fixture) and assert Rust decompresses it identically.
- TTL respected (mockable in tests; integration in =wrangler dev=).

** entry.rs (integration, via =wrangler dev=)
Run Rust worker locally, issue HTTP requests:
- =GET /= returns 200 + =text/html= + contains known marker string.
- =GET /chart?city=...= returns 200 + =image/svg+xml= + non-empty body.
- =GET /forecast?...= returns 200 + =application/json= + schema matches.
- Bad params → 4xx, not 5xx.
- Missing city → same error shape as Python worker (capture current behavior first).
- Deeplink parsing: 10 fixture URLs; response byte-equal against Python worker running on another port.
- KV cache hit path: second request faster AND returns same bytes.

** Parity harness (end-to-end)
=tools/parity/diff.py=:
- Spawns =wrangler dev= for Python on :8787 and Rust on :8788.
- Hits ~200 randomized parameter sets against both.
- Reports SVG diff, JSON diff, status-code diff.
- Exits non-zero on any mismatch. Run in CI on every PR.

** Performance tests (post-parity)
- =criterion= micro-benches for: CSV parse (per 1MB), SVG render per
  chart, Ridge fit per call. Commit baseline numbers; fail if a PR
  regresses >20%.
- Load test with =oha= or =vegeta=: 100 RPS for 60s against local
  Rust worker. Record p50/p95/p99 CPU time from =wrangler tail=.

** Suggested ordering (smallest shippable chunks)
1. Fixtures + =israel_time= + tests. ½ day. Green CI before moving on.
2. =city_translations= + generator. ½ day.
3. =data_loading= + fixture parity. 1 day. Merge.
4. =forecast= non-Ridge models + tests. ½ day. Merge.
5. =forecast= Ridge + intermediates pinned. 1 day. Merge.
6. =svg.rs= component tests first, then golden. 2 days. Merge behind flag.
7. =kv_cache= + =entry.rs= routes one at a time (=/healthz=, =/=, =/chart=, =/forecast=). 1 day.
8. Parity harness in CI. ½ day.
9. Load test + cutover.

Each merge should leave =main= shippable: Rust worker deploys to
=alarms-rs= route, Python worker stays at =alarms=.

* Cutover Checklist

- [ ] All parity tests green (SVG byte-equal, forecasts ≤1e-4 drift).
- [ ] Load test: 100 RPS for 60s, p99 < 200ms (Python baseline ~700ms).
- [ ] Memory: peak per request < 16MB (was ~80MB on Python).
- [ ] Bundle size < 2MB compressed.
- [ ] KV cache keys either reused (verified compatible) or prefixed and old keys left to TTL out.
- [ ] DNS/route swap: =alarms.metta.workers.dev= points to Rust worker.
- [ ] Python worker kept deployed for 7 days under =alarms-py.metta.workers.dev= as rollback.
- [ ] Delete =worker/= directory and update =ARCHITECTURE.org=.

* Open Questions for the Implementer

1. Is the SVG numeric formatting in =alarms_core.py= already
   deterministic across Python versions? If not, the parity test
   needs a tolerance comparator (parse SVG, compare path commands
   numerically) instead of byte-equality.
2. Does the client JS depend on any =JsProxy=-specific behavior, or
   purely on HTTP responses? (Expected: purely HTTP — verify.)
3. Is the existing zlib-compressed CSV in KV worth preserving, or
   simpler to invalidate and let it repopulate? Recommendation:
   invalidate via key prefix change.
4. Ridge =alpha=10= and 34-feature design matrix — confirm feature
   order is fully specified in =forecast.py= so Rust matches column-
   for-column.
